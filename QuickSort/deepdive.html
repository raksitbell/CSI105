How the Fixed Pivot Quick Sort Works

This page details the specific variation of the `asc` Quick Sort logic utilized in this visualizer mapping round rules against Pointers R and F.

⚡ Divide and Conquer

Quick Sort relies heavily on determining a correct 'Pivot' and then actively walking two pointer bounds across a subset to throw variables above or below the lock mark. This specific workshop tracks where the Pivot initially resides, traversing the opposite vector and triggering swap exchanges based on Ascending rules when boundaries inevitably collide.

1. Arrays & Queue Initialization

Before evaluating anything, the total length mapped defines the initial partition. The queue system ensures parallel processing runs step by step.

// From script.js — Queue setup for frames
let queue = [
  {
    low: 0,
    high: arr.length - 1,
    R: 0,
    F: arr.length - 1,
    pivot_pos: arr.length - 1, // Default Fixed Pivot is right-most Element
  },
];

2. Partition Checking & Moving Pointers

Each step, unless the points meet (`R == F`), we run the check conditions. If the pivot sits on `F`, we assess against `R`. If validation succeeds, `R` simply advances.

// From script.js — Process Tasks Step Logic
if (pivot_pos === F) {
  if (arr[R] < arr[F]) {
    task.R += 1; // R is smaller, keep checking next
  } else if (arr[R] === arr[F]) {
    task.R += 1; // Resolve exact match logic by scanning further
  } ...

3. Target Hits - Swap Trigger

When a value is discovered to be out-of-order `arr[R] > arr[F]`, a swap evaluates and the pivot location implicitly jumps pointers.

if (arr[R] > arr[F]) {
  // Standard Swap Mechanics
  let temp = arr[R];
  arr[R] = arr[F];
  arr[F] = temp;
  
  // Pivot shifted; so lock pivot to R, and decrement F
  task.pivot_pos = R;
  task.F -= 1;
}

4. Round Resolution & Recursion Setup

When `R` & `F` finally satisfy `R == F`, the current subset ends. Any adjacent partitions of length greater than 1 become appended to the next queue batch loop.

// From script.js — Next Round Checking
let resolved_pivot = task.R; // R == F here

// Check for valid bounds on Left partition
if (resolved_pivot - 1 > task.low) {
  queue.push({ low: task.low, high: resolved_pivot - 1, R: task.low, F: resolved_pivot - 1, ... });
}

// Check for valid bounds on Right partition
if (resolved_pivot + 1 < task.high) {
  queue.push({ low: resolved_pivot + 1, high: task.high, R: resolved_pivot + 1, F: task.high, ... });
}
