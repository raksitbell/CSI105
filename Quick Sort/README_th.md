# เวิร์กชอป Quick Sort แบบโต้ตอบ

ยินดีต้อนรับสู่โปรเจกต์ Quick Sort ใน Data Structures Workshop! โฟลเดอร์นี้มีโปรแกรมสร้างภาพ (Visualizer) แบบโต้ตอบสำหรับอัลกอริทึม Quick Sort ซึ่งออกแบบมาโดยเฉพาะเพื่อแสดงให้เห็นถึงการทำงานของ Quick Sort แบบ parallel และมีการกำหนด pivot ไว้ที่ตำแหน่งสุดท้ายอย่างชัดเจน

## ภาพรวม
Quick Sort เป็นอัลกอริทึมการเรียงลำดับแบบแบ่งแยกและเอาชนะ (divide-and-conquer) ที่มีประสิทธิภาพสูงทำงานโดยการเลือกองค์ประกอบที่เป็น 'จุดหมุน' (pivot) จากอาร์เรย์ และแบ่งองค์ประกอบอื่นๆ ออกเป็นสองอาร์เรย์ย่อยตามข้อกำหนดว่าพวกมันมีค่าน้อยกว่าหรือมากกว่า pivot หรือไม่ จากนั้นอาร์เรย์ย่อยจะถูกเรียงลำดับซ้ำไปเรื่อยๆ

การนำไปใช้งานนี้แสดงภาพของขั้นตอนการทำงานแบบทีละขั้น ช่วยให้คุณเห็นได้อย่างชัดเจนว่า `Pointer R` และ `Pointer F` ทำการสแกนพาร์ติชันและสลับค่าอย่างไร

## วิธีการใช้งาน
1. เปิดไฟล์ `index.html` ในเว็บเบราว์เซอร์ของคุณ
2. ป้อนอาร์เรย์ของตัวเลขโดยคั่นด้วยเครื่องหมายจุลภาคในช่องกรอกข้อมูล หรือคลิกปุ่ม **Random** เพื่อสร้างชุดข้อมูลแบบสุ่ม
3. คลิก **Start Quick Sort** เพื่อเริ่มต้นกระบวนการ
4. ใช้ปุ่มควบคุมการเล่น (**Play**, **Pause**, **Next**, **Prev**) เพื่อเลื่อนดูแต่ละขั้นตอนของอัลกอริทึม
5. ใช้ปุ่ม **1x Speed** / **2x Speed** เพื่อควบคุมความเร็วในการเล่นภาพเคลื่อนไหว
6. ตรวจสอบกล่อง **Execution Log** ใต้แอนิเมชันเพื่ออ่านเงื่อนไขและการกระทำที่เกิดขึ้นอย่างแม่นยำในแต่ละขั้นตอน
7. คลิก **Skip Round** เพื่อข้ามการสแกนทีละสเตปของพอยน์เตอร์และกระโดดตรงไปยังสถานะสิ้นสุดของรอบพาร์ติชันนั้นๆ โดยตรง

## กฎ "เรียงจากน้อยไปมาก" (ยึด Pivot ที่ดัชนีสุดท้าย)
โปรแกรมแบบจำลองของเราจะติดตาม Pivot ไว้ที่ดัชนีสุดท้ายของกรอบเขตพาร์ติชันที่ยังไม่ได้เรียงลำดับเสมอ
ในแต่ละขั้นตอน `Pointer R` และ `Pointer F` อาจอยู่ในตำแหน่ง Pivot อย่างใดอย่างหนึ่งขึ้นอยู่กับการเลื่อนผ่าน:

*   **เมื่อ Pivot อยู่ที่ (F):** 
    ถ้าค่าที่ R > ค่าที่ F: จะสลับค่าระหว่าง R กับ F (Swap(R,F)) และขยับ F ไปซ้าย (`F -= 1`)
    ถ้าไม่ใช่: จะขยับ R เดินหน้า (`R += 1`)
*   **เมื่อ Pivot อยู่ที่ (R):**
    ถ้าค่าที่ R > ค่าที่ F: จะสลับค่าระหว่าง R กับ F (Swap(R,F)) และขยับ R ไปขวา (`R += 1`)
    ถ้าไม่ใช่: จะขยับ F ถอยหลัง (`F -= 1`)

เมื่อ `R` ชนกับ `F` รอบพาร์ติชันจะสิ้นสุดลง หมายความว่า Pivot ถูกล็อคในตำแหน่งที่ถูกต้องแล้ว และอาร์เรย์จะถูกแบ่งออกเป็นอาร์เรย์ย่อยใหม่เพื่อดำเนินการตามขั้นตอนต่อไป

## สร้างด้วย
*   HTML ดั้งเดิม, JavaScript, และตกแต่งด้วย Tailwind CSS (ใช้ฟอนต์ `Kanit`)
*   ไม่ได้ใช้ UI framework หนักๆ ใดๆ เลย พึ่งพาเพียงการดัดแปลง DOM เท่านั้น
