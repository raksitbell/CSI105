Step-by-Step: How The Code Works

Build the Stack Class

A Stack uses a simple array internally. push() adds to the end and pop() removes from the end ‚Äî LIFO. top() peeks at the top without removing. 

// From script.js ‚Äî The Stack class
class Stack {
    constructor() { this.items = []; }

    push(val)  { this.items.push(val); }
    pop()      { return this.items.pop(); }
    top()      { return this.items[this.items.length - 1]; }
    clear()    { this.items = []; }
    isEmpty()  { return this.items.length === 0; }
}
üí° Think of it like this: A stack of books ‚Äî you can only add or remove from the top.

Build the Queue Class

A Queue removes from the front using shift(). enqueue() adds to the rear, dequeue() removes from the front ‚Äî FIFO. front() and rear() peek at both ends. 

// From script.js ‚Äî The Queue class
class Queue {
    constructor() { this.items = []; }

    enqueue(val) { this.items.push(val); }
    dequeue()    { return this.items.shift(); }
    front()      { return this.items[0]; }
    rear()       { return this.items[this.items.length - 1]; }
    clear()      { this.items = []; }
    isEmpty()    { return this.items.length === 0; }
}
üí° Think of it like this: A queue at a ticket booth ‚Äî the first person in line gets served first.

Push (Stack) vs Enqueue (Queue)

Both add an item. The addToBoth() function lets you add the same value to both structures at once to compare behavior. 

// From script.js ‚Äî add to both
window.addToBoth = () => {
    const val = getInput();
    if (!val) return;

Pop (Stack) vs Dequeue (Queue)

pop() removes from the top (last added), while dequeue() removes from the front (first added). Same data, different removal order. 

// From script.js ‚Äî removing items
window.popStack = () => {
    if (stack.isEmpty()) return showToast("Stack Empty", "error");
    stack.pop();    // removes TOP
    save(); render();
};

window.dequeueQueue = () => {
    if (queue.isEmpty()) return showToast("Queue Empty", "error");
    queue.dequeue(); // removes FRONT
    save(); render();
};

Peek ‚Äî Look Without Removing

top() shows the stack's top, front() the queue's first item, rear() the last. None remove the item. 

// From script.js ‚Äî peek operations
window.peekStack = () =>
    stack.isEmpty()
        ? showToast("Stack Empty", "error")
        : showToast(`Top: ${stack.top()}`);

window.peekQueueFront = () =>
    queue.isEmpty()
        ? showToast("Queue Empty", "error")
        : showToast(`Front: ${queue.front()}`);

window.peekQueueRear = () =>
    queue.isEmpty()
        ? showToast("Queue Empty", "error")
        : showToast(`Rear: ${queue.rear()}`);

Persistence with localStorage

Every operation saves data to localStorage. On page reload, load() restores the saved state into both classes. 

// From script.js ‚Äî save & load
const STORAGE_KEY = "sq_data";

const save = () => {
    localStorage.setItem(STORAGE_KEY, JSON.stringify({
        stack: stack.getAll(),
        queue: queue.getAll()
    }));
};

const load = () => {
    const data = JSON.parse(localStorage.getItem(STORAGE_KEY));
    if (data) {
        stack.load(data.stack || []);
        queue.load(data.queue || []);
    }
    render();
};
üí° Why this works: getAll() returns a plain array copy which can be serialized to JSON. On reload, load() restores it back.
