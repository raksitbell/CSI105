Step-by-Step: How The Code Works

Build the Stack Class

A Stack uses a simple array internally. push() adds to the end and pop() removes from the end ‚Äî this makes the last item added always the first to leave (LIFO). top() peeks at the top item without removing it. 

// From script.js ‚Äî The Stack class
class Stack {
    constructor() { this.items = []; }

    push(val)  { this.items.push(val); }
    pop()      { return this.items.pop(); }
    top()      { return this.items[this.items.length - 1]; }
    clear()    { this.items = []; }
    isEmpty()  { return this.items.length === 0; }
}
üí° Think of it like this: A stack of books on a desk ‚Äî you can only add or remove from the top.

Build the Queue Class

A Queue also uses an array, but it removes from the front using shift(). enqueue() adds to the rear, dequeue() removes from the front ‚Äî first in, first out (FIFO). It also provides front() and rear() to peek at both ends. 

// From script.js ‚Äî The Queue class
class Queue {
    constructor() { this.items = []; }

    enqueue(val) { this.items.push(val); }
    dequeue()    { return this.items.shift(); }
    front()      { return this.items[0]; }
    rear()       { return this.items[this.items.length - 1]; }
    clear()      { this.items = []; }
    isEmpty()    { return this.items.length === 0; }
}
üí° Think of it like this: A queue at a ticket booth ‚Äî the first person in line gets served first.

Push (Stack) vs Enqueue (Queue)

Both operations add an item, but the naming is different to reflect the semantics. In this project, the addToBoth() function lets you add the same value to both the stack and queue at once to compare behavior. 

// From script.js ‚Äî add to both structures
window.addToBoth = () => {
    const val = getInput();
    if (!val) return;
    stack.push(val);       // adds to the TOP
    queue.enqueue(val);    // adds to the REAR
    save(); render();
};

Pop (Stack) vs Dequeue (Queue)

This is where the key difference shows. pop() removes from the top (the last item added), while dequeue() removes from the front (the first item added). Same data, different removal order. 

// From script.js ‚Äî removing items
window.popStack = () => {
    if (stack.isEmpty()) return showToast("Stack Empty", "error");
    stack.pop();    // removes from the TOP (last added)
    save(); render();
};

window.dequeueQueue = () => {
    if (queue.isEmpty()) return showToast("Queue Empty", "error");
    queue.dequeue(); // removes from the FRONT (first added)
    save(); render();
};

Peek ‚Äî Look Without Removing

top() shows the top of the stack, front() shows the first item in the queue, and rear() shows the last. None of these remove the item ‚Äî they just look at it. 

// From script.js ‚Äî peek operations
window.peekStack = () =>
    stack.isEmpty()
        ? showToast("Stack Empty", "error")
        : showToast(`Top: ${stack.top()}`);

window.peekQueueFront = () =>
    queue.isEmpty()
        ? showToast("Queue Empty", "error")
        : showToast(`Front: ${queue.front()}`);

window.peekQueueRear = () =>
    queue.isEmpty()
        ? showToast("Queue Empty", "error")
        : showToast(`Rear: ${queue.rear()}`);

Persistence with localStorage

Every time you push, pop, enqueue, or dequeue, the data is saved to localStorage. When the page reloads, it loads the saved data back into the Stack and Queue classes using load(). 

// From script.js ‚Äî save & load
const STORAGE_KEY = "sq_data";

const save = () => {
    localStorage.setItem(STORAGE_KEY, JSON.stringify({
        stack: stack.getAll(),
        queue: queue.getAll()
    }));
};

const load = () => {
    const data = JSON.parse(localStorage.getItem(STORAGE_KEY));
    if (data) {
        stack.load(data.stack || []);
        queue.load(data.queue || []);
    }
    render();
};
üí° Why this works: getAll() returns a plain array copy, which can be serialized to JSON. On reload, load() restores the array back into the class.
