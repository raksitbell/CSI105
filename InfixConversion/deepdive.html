🧠 What is the Shunting Yard Algorithm?

The Shunting Yard algorithm, invented by Edsger Dijkstra, is a method for parsing mathematical expressions specified in infix notation. It can be used to produce output in Reverse Polish Notation (Postfix) or Prefix notation. The algorithm uses a stack to hold operators and pushes operands directly to the output.

Infix:
A
+
B
→
Postfix:
A
B
+ ✨

1. Tokenization

Before converting the expression, we need to break the input string into a list of meaningful components called "tokens" (numbers, letters, operators, and parentheses). We use regular expressions for this.

// From script.js — Tokenizer helper
const tok = (e) => e.match(/[a-zA-Z0-9]+|[\+\-\*\/\^\(\)]/g) || [];
const isOp = (t) => /^[a-zA-Z0-9]+$/.test(t);

2. Processing Loop Setup

We iterate through each token. If it's an operand, it goes straight to the output. If we are converting to prefix, we must reverse the tokens and flip the parentheses first.

// From script.js — Setup & Prefix handling

// If Prefix: reverse tokens and swap parenthesis directions

  if (isOp(c))
    out.push(c); // Append operand to output

3. Handling Operators by Precedence

When an operator is found, we pop operators from the stack to the output while the stack's top has higher or equal precedence. Notably, we handle right-associativity (like exponentiation ^) differently for Prefix vs Postfix.

// From script.js — Precedence checks
while (ops.length && ops[ops.length - 1] !== "(") {
  let top = ops[ops.length - 1];
  
  let cond = pre
    ? p[top] > p[c] || (p[top] === p[c] && c === "^")
    : p[top] > p[c] || (p[top] === p[c] && c !== "^");
    
  if (cond) out.push(ops.pop());
  else break;
}
ops.push(c); // Push current operator
💡

Associativity dictates the direction of evaluation: Left-associative operators (+, -, *, /) evaluate left-to-right, while right-associative ones (^) evaluate right-to-left. The strictness of the condition p[top] === p[c] determines this.

4. Final Polish & Output

Finally, we flush any remaining operators from the stack to the output array. If we were converting to Prefix, we must reverse the final array so it reads correctly.

// From script.js — Finalization
while (ops.length) out.push(ops.pop());

// If Prefix: restore original order by reversing again
return (pre ? out.reverse() : out).join(" ");
