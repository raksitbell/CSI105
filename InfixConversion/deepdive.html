🧠 What is the Shunting Yard Algorithm?

The Shunting Yard algorithm, invented by Edsger Dijkstra, is a method for parsing mathematical expressions specified in infix notation. It can be used to produce output in Reverse Polish Notation (Postfix) or Prefix notation. The algorithm uses a stack to hold operators and pushes operands directly to the output.

Infix:
A
+
B
→
Postfix:
A
B
+ ✨

1. Tokenization & Input Prep

First, the user's input string is broken down into a list of meaningful tokens (operands, operators, parentheses) using a regular expression. This gives us an array of items to iterate through.

// From script.js — Tokenizer helper
const tok = (e) => e.match(/[a-zA-Z0-9]+|[\+\-\*\/\^\(\)]/g) || [];

let t = tok(exp);

2. Prefix Pre-processing (Optional)

If the user selected "Prefix" conversion, we actually process the expression backwards. We reverse the token array and swap left and right parentheses, then run the standard algorithm.

// From script.js — Reverse tokens & swap parenthesis
if (pre) {
  t = t.reverse().map((c) => (c === "(" ? ")" : c === ")" ? "(" : c));
}

3. The Core Loop: Operands & Parentheses

We initialize an out array for results and an ops stack for operations. As we iterate over each token:
• Operands (letters/numbers) are pushed directly to the output.
• Open Parentheses ( are pushed to the stack.
• Close Parentheses ) trigger popping from the stack to the output until the matching open parenthesis is found.

let out = [], ops = [];
t.forEach((c) => {
  if (isOp(c)) out.push(c); // Operand straight to output
  else if (c === "(") ops.push(c); // Open parenthesis to stack
  else if (c === ")") {
    // Pop until we discover '('
    while (ops.length && ops[ops.length - 1] !== "(") 
      out.push(ops.pop());
    ops.pop(); // Discard the '(' itself
  }

4. The Core Loop: Operators & Precedence

If the token is an operator (like +, *, ^), we check its precedence against the operator at the top of the stack. Because Prefix evaluates right-to-left for powers (associativity), its conditional check differs slightly from Postfix. If the top operator has higher (or equal) precedence, we pop it to the output first.

else {
  while (ops.length && ops[ops.length - 1] !== "(") {
    let top = ops[ops.length - 1];
    // Check precedence & associativity (Prefix handles ^ differently)
    let cond = pre
      ? p[top] > p[c] || (p[top] === p[c] && c === "^")
      : p[top] > p[c] || (p[top] === p[c] && c !== "^");
      
    if (cond) out.push(ops.pop());
    else break;
  }
  ops.push(c); // Finally push the new operator
}

5. Final Polish & Output

Once the loop is finished, we empty any remaining operators from the stack into the output array. If we were executing Prefix conversion, this is the final moment where we reverse the entire output array back to normal so it reads correctly. The user sees the final joined string on the screen!

// Flush remaining operators to the output
while (ops.length) out.push(ops.pop());

// Reverse back if prefix, join array elements with space
return (pre ? out.reverse() : out).join(" ");
